Команды Git
Команда Описание
git init - Эта команда создаст .git репозиторий в Вашем проекте
git clone url -Если у Вас отсутствует проект на личном или рабочем компьютере, то эта команда поможет клонировать/загрузить весь проект в текущую директорию.
git add file -Если Вы хотите добавить все файлы с расширением .html в staging area то эта команда отлично подойдет.
git add . -Если Вы хотите добавить всё из директории проекта в staging area, то эта команда сделает всё сама.
git commit -m "comment" -Берёт файлы из staging area и “фиксирует” их в Ваш локальный репозиторий.
git commit -a -m "comment" - Чтобы зарегистрировать добавление вышеприведенных файлов необходимо выполнить следующую команду –a означает: добавить все изменения в индекс до передачи. -m : сообщение
git push url -Загружает все изменения локальной ветки в удалённый репозиторий
git status -Покажет что уже было добавлено в staging area и какие файлы были изменены и ждут перемещения в staging area.
git log -История коммитов для текущей ветки
git branch branch_name -Создает сущность, называемую branch(ветвь). Ветвь - это точная копия Ваших файлов.
git checkout branch_name -Позволит Вам переключить контроль над созданной Вами веткой и работать в её пределах.
git checkout -b branch_name -делать с текущего места новую ветку и сразу же продолжить с ней
git remote -Выведет список из всех удалённых репозиториев, которые были добавлены к Вашему проекту.
